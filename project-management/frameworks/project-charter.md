# 프로젝트 헌장 (Project Charter) 가이드

## 개요

프로젝트의 **존재 이유와 권한을 공식적으로 승인**하는 문서.
비유: 항해를 떠나는 선장에게 왕이 내려주는 **임명장 및 항해 명령서**.

## 핵심 구성 요소

### 1. 프로젝트 배경 및 목적 (Why)

- **비즈니스 케이스**: 왜 지금 이 프로젝트를 해야 하는가? (기회/문제)
- **목표 (SMART)**: 구체적이고 측정 가능한 성공 기준
  - 예: 3개월 내 가입 전환율 5% → 8% 달성

### 2. 범위 (Scope)

- **In-Scope**: 프로젝트에 포함되는 업무 및 기능
- **Out-of-Scope**: 명확히 제외되는 사항 (범위 확산 방지)

### 3. 주요 이해관계자 (Stakeholders)

- **Sponsor**: 예산 승인 및 최종 의사결정자
- **PM**: 프로젝트 총괄 책임자
- **Team**: 실무 수행 조직
- **Customer**: 최종 결과물 사용자

### 4. 제약 조건 및 가정 (Constraints & Assumptions)

- **제약**: 반드시 지켜야 할 조건 (예산 $10k, 마감일 12/31, 법적 규제)
- **가정**: 사실이라고 전제하는 상황 (예: 외부 API가 9월에 오픈된다)

### 5. 마일스톤 (High-level Timeline)

| 마일스톤  | 예상 일정  | 주요 산출물     |
| :-------- | :--------- | :-------------- |
| 기획 완료 | YYYY-MM-DD | 기획서, UI 시안 |
| 개발 착수 | YYYY-MM-DD | 개발 환경 셋업  |
| 베타 오픈 | YYYY-MM-DD | MVP 배포        |

## 승인 (Sign-off)

프로젝트 헌장에 서명함으로써 프로젝트가 공식적으로 시작됨을 선언합니다.
