# 위협 모델링 (Threat Modeling) 가이드

## 개요

시스템에 대한 잠재적 위협을 **체계적으로 식별**하고 대응 방안을 설계합니다.
비유: 집에 도둑이 들어올 수 있는 **모든 경로(문, 창문, 지하실)**를 점검하는 것.

## STRIDE 모델

| 위협 유형                  | 설명                        | 대응 원칙                |
| :------------------------- | :-------------------------- | :----------------------- |
| **S**poofing               | 다른 사용자/시스템으로 위장 | 강력한 인증 (MFA)        |
| **T**ampering              | 데이터/코드 무단 변조       | 무결성 검증 (해시, 서명) |
| **R**epudiation            | 행위 부인 (내가 안 했다)    | 감사 로그, 부인방지      |
| **I**nfo Disclosure        | 정보 유출                   | 암호화, 접근 제어        |
| **D**enial of Service      | 서비스 거부 공격            | 속도 제한, CDN, 스케일링 |
| **E**levation of Privilege | 권한 상승                   | 최소 권한 원칙, RBAC     |

## 분석 절차

### 1. 자산 식별

- 보호 대상 데이터 (개인정보, 결제, 영업비밀)
- 핵심 시스템 (인증, DB, API, 관리자 패널)
- 인프라 (서버, 네트워크, 클라우드)

### 2. 진입점(Entry Point) 매핑

- 외부 API 엔드포인트
- 웹 UI 입력 필드
- 파일 업로드 기능
- 관리자 인터페이스
- 제3자 연동 (OAuth, Webhook)

### 3. 위협 시나리오 도출 (STRIDE 적용)

| 자산   | 진입점   | STRIDE 유형   | 위협 시나리오     | 심각도                   |
| :----- | :------- | :------------ | :---------------- | :----------------------- |
| [자산] | [진입점] | [S/T/R/I/D/E] | [구체적 시나리오] | Critical/High/Medium/Low |

### 4. 대응 방안 매핑

각 위협에 대한 기술적/정책적 대응 조치 수립

## 주의사항

- 위협 모델은 **지속적으로 업데이트** (시스템 변경 시마다)
- **공격자 관점**에서 사고할 것
- 기술적 위협뿐 아니라 **사회공학적 공격**도 고려
