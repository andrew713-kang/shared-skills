# UX 리서치 가이드

## 개요

사용자의 **진짜 니즈와 행동 패턴**을 발견하는 체계적 탐구.
비유: 의사가 환자에게 "어디가 아프세요?"가 아니라 **증상을 관찰하고 검사하는 것**.

## 리서치 유형

| 유형   | 목적        | 방법                        | 시점           |
| :----- | :---------- | :-------------------------- | :------------- |
| 탐색적 | 문제 발견   | 인터뷰, 관찰, 컨텍스트 조사 | 기획 초기      |
| 평가적 | 솔루션 검증 | 사용성 테스트, A/B 테스트   | 디자인/개발 중 |
| 정량적 | 패턴 확인   | 설문, 분석 도구(GA)         | 지속적         |

## 사용자 인터뷰 가이드

### 질문 설계 원칙

- ❌ "이 기능 좋아요?" (유도 질문)
- ✅ "이 작업을 마지막으로 했을 때를 말씀해주세요" (행동 기반)
- ✅ "그 때 가장 어려웠던 점은?" (페인 포인트 탐색)
- ✅ "이상적으로는 어떻게 되면 좋겠어요?" (니즈 발견)

### 인터뷰 구조 (45분)

1. **도입** (5분): 자기소개, 목적 설명, 동의
2. **맥락** (10분): 배경, 현재 사용 도구/방법
3. **심층** (20분): 핵심 태스크, 페인 포인트, 니즈
4. **탐색** (5분): 이상적 경험, 미충족 요구
5. **마무리** (5분): 추가 의견, 감사

## 벤치마킹 (Heuristic Evaluation)

Jakob Nielsen의 10가지 휴리스틱:

| #   | 원칙               | 점검 내용             |
| :-- | :----------------- | :-------------------- |
| 1   | 시스템 상태 가시성 | 현재 상태/진행 표시   |
| 2   | 실제 세계와 매칭   | 사용자 언어/멘탈모델  |
| 3   | 사용자 제어와 자유 | 실행 취소, 뒤로가기   |
| 4   | 일관성과 표준      | 플랫폼 컨벤션 준수    |
| 5   | 에러 방지          | 실수 방지 설계        |
| 6   | 기억보다 인식      | 옵션/정보 가시화      |
| 7   | 유연성과 효율      | 초보↔전문가 모두 지원 |
| 8   | 미적이고 미니멀    | 불필요 정보 제거      |
| 9   | 에러 회복 지원     | 명확한 에러 메시지    |
| 10  | 도움말과 문서      | 쉬운 접근성           |
