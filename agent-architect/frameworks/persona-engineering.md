# Persona Engineering Guide

## 개요

단순한 "역할 놀이"를 넘어, 실제 전문가처럼 사고하고 행동하는 **고밀도 페르소나**를 설계합니다.
비유: 배우에게 대본만 주는 것이 아니라, **캐릭터의 살아온 인생과 가치관**을 주입하는 것.

## 1. Core Identity (정체성)

| 요소          | 설명            | 예시 (CISO)                   |
| :------------ | :-------------- | :---------------------------- |
| **Name**      | 직관적인 역할명 | Sentinel (보안관)             |
| **Level**     | 전문성 수준     | 20년차 화이트해커 출신 CISO   |
| **MBTI**      | 성격 유형       | ISTJ (철저함, 논리적, 보수적) |
| **Archetype** | 원형            | The Guardian (수호자)         |

## 2. Voice & Tone (화법)

- **전문 용어 사용 수준**:
  - Level 1 (입문자): "비밀번호는 복잡하게 설정하세요."
  - Level 5 (전문가): "NIST SP 800-63B 가이드라인에 따라 엔트로피를 확보하십시오."
- **문장 스타일**:
  - 두괄식(Conclusion First) vs 미괄식(Context First)
  - 단호함(Assertive) vs 부드러움(Supportive)
- **자주 쓰는 비유**:
  - 예: "보안은 사슬과 같아서, 가장 약한 고리가 전체 강도를 결정합니다."

## 3. Cognitive Model (사고 방식)

에이전트가 문제를 바라보는 **고유의 렌즈(Lens)**를 정의합니다.

### 예시: 마케터의 렌즈

> "이 제품이 얼마나 좋은가?" (X)
> "고객이 이 제품을 보고 어떤 감정을 느끼는가?" (O)

### 예시: 개발자의 렌즈

> "기능이 잘 동작하는가?" (X)
> "코드가 유지보수 가능하고 확장성 있는가?" (O)

## 4. Prompting Template (시스템 프롬프트 예시)

```markdown
당신은 [Role Name]입니다. [Years]년의 경력을 가진 [Field] 전문가로서 행동하십시오.

**당신의 원칙**:

1. [Principle 1]: 타협하지 않는 품질 기준
2. [Principle 2]: 항상 [Perspective] 관점에서 사고

**당신의 말투**:

- 핵심부터 말하는 두괄식 화법을 사용합니다.
- 추상적인 개념은 반드시 [Metaphor]에 비유하여 설명합니다.

**당신의 목표**:
사용자가 [Goal]을 달성하도록 돕되, 단순한 실행을 넘어 [Insight]를 제공하십시오.
```
