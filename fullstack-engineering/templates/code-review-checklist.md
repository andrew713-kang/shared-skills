# 코드 리뷰 체크리스트 (Code Review Checklist)

**Reviewer**: [이름] | **Target**: [PR 번호/브랜치]

## 1. 기능 동작 (Functionality)

- [ ] 요구사항(기획서/Jira)을 모두 충족하는가?
- [ ] 엣지 케이스(Edge Case)를 고려했는가? (예: 입력값 없음, 최대값, 네트워크 실패)
- [ ] 기존 기능에 부작용(Side Effect)은 없는가?

## 2. 가독성 및 유지보수성 (Readability)

- [ ] 변수명/함수명은 의도를 명확히 드러내는가? (`d` vs `daysSinceLogin`)
- [ ] 함수 하나가 하나의 역할만 하는가? (단일 책임 원칙)
- [ ] 중복 코드는 없는가? (DRY 원칙)
- [ ] 주석은 '무엇'이 아닌 **'왜'**를 설명하는가?
- [ ] 매직 넘버/스트링을 사용하지 않았는가? (상수화 필요)

## 3. 아키텍처 및 디자인 패턴

- [ ] 계층 간 의존성 규칙을 지켰는가? (Controller → Service → Repository)
- [ ] 적절한 디자인 패턴을 사용했는가? (Strategy, Factory, Singleton 등)
- [ ] 전역 상태(Global State) 사용을 최소화했는가?

## 4. 성능 (Performance)

- [ ] N+1 쿼리 문제는 없는가?
- [ ] 불필요한 루프나 연산은 없는가?
- [ ] 메모리 누수 가능성은 없는가? (이벤트 리스너 해제 등)

## 5. 보안 (Security)

- [ ] SQL Injection / XSS 방지 처리가 되어있는가?
- [ ] 민감 정보(API Key, 비밀번호)가 로그나 코드에 노출되지 않았는가?
- [ ] 권한 체크(Authorization)가 누락되지 않았는가?

## 6. 테스트 (Testing)

- [ ] 핵심 로직에 대한 단위 테스트가 포함되었는가?
- [ ] 테스트 케이스가 다양한 시나리오를 커버하는가?
